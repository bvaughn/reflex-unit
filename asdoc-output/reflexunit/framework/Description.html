Packagereflexunit.framework
Classpublic class Description

Describes a single class and the testable methods it defines. A Description analyzes the provided class to determine which methods are considered testable (see below). Each testable method is bundled into a MethodModel for later use. There are several possible ways for a method to be determined testable: The first way is for the provided test class to define a static accessor named testableMethods. Such a method should return an Array of Function references. That Array then defines the entire set of testable methods. If such an accessor is not defined, the provided test will be analyzed using the IntrospectionUtil. In this case all methods meeting one of the following conditions will be considered testable:
  • Method name begins with "test", accepts no parameters, and has a return type of void
  • Method is markd with the metadata tag "Test"

A Description may only be run (ie. tested) once. Because of this, a Description should in most cases be cloned before being passed to a Runner.

allowParallelAsynchronousTests:Boolean  [read-only]

Current test has elected to allow parallel execution of its asynchronous test methods. If FALSE, no other tests should be run until this method has completed its assertions. (In that event all previously executed asynchronous tests should be allowed to complete before this method is started.)
